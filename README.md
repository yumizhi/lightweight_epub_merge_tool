# Lightweight EPUB Merge Tool

Browser-local EPUB merge, split, and inspect tool.

The web UI runs entirely in the browser. EPUB files are kept in the current browser session and are processed with a Web Worker, so files do not need to be uploaded to a server.

## Features

- Merge multiple EPUB files into one EPUB
- Preserve volume-style or flat table-of-contents structure
- Automatically infer book order from EPUB metadata/title patterns
- Optional input-order mode for manual ordering
- Split EPUBs generated by this tool back into source files
- Inspect generated EPUB merge manifests
- English and Chinese UI
- Desktop, tablet, and mobile web layouts

## Local Development

Install dependencies:

```bash
npm install
```

Start the web UI:

```bash
npm run dev:web
```

Then open the URL shown by Vite, usually:

```text
http://127.0.0.1:5173/
```

Build the web app:

```bash
npm run build:web
```

Run TypeScript checks and the web build:

```bash
npm run verify
```

## GitHub Pages Deployment

This repository includes a manual GitHub Pages workflow:

```text
.github/workflows/deploy-pages.yml
```

It does not deploy on every push. To deploy:

1. Open the repository on GitHub.
2. Go to Actions.
3. Select "Deploy Web UI to GitHub Pages".
4. Click "Run workflow".

The workflow builds `web/` with Vite and publishes `dist-web/` to GitHub Pages.

If this is the first Pages deployment for the repository, enable Pages with GitHub Actions as the source in repository settings.

## Project Layout

```text
ts/src/      EPUB processing core
web/src/     React web UI
web/         Vite app configuration
```

## Privacy

Files are processed locally in the browser session. The GitHub Pages deployment only serves static JavaScript, CSS, and HTML.
